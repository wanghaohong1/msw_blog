# 后端技术栈导览

后端开发的核心价值在于支撑业务稳定运行、保证数据可靠存储并提供可扩展的服务能力。本文从能力地图、学习路径与实践方法三个角度梳理主流后端技术栈，帮助你快速建立全局视角。

## 能力地图

| 领域 | 关键要点 | 常用技术栈 |
| --- | --- | --- |
| 语言与运行时 | 熟悉主流语言语法、并发模型、内存管理 | Java、Kotlin、Go、Rust、Node.js |
| Web 框架 | 构建 HTTP API、过滤器、MVC、AOP | Spring Boot、Spring Cloud、Express、Gin |
| 数据存储 | 事务/索引/锁、分片、读写分离 | MySQL、PostgreSQL、Redis、MongoDB、ElasticSearch |
| 消息与异步 | 应急扩容、削峰填谷、解耦 | Kafka、RabbitMQ、RocketMQ |
| 服务治理 | 注册发现、配置、熔断限流、链路追踪 | Nacos、Eureka、Sentinel、SkyWalking、Zipkin |
| 安全与鉴权 | 会话管理、令牌、权限模型、数据脱敏 | Spring Security、OAuth2、JWT、Shiro |
| 交付与运维 | 构建、发布、监控、日志、容器编排 | Docker、Kubernetes、Jenkins、Argo CD、Prometheus、Grafana |

## 学习路径建议

1. **打基础（1-2 个月）**
   - 精通一门后端主语言，理解集合、并发、异常、泛型等核心语法。
   - 搭建最小可用服务：完成 RESTful API、数据库 CRUD、基本日志。
   - 掌握 Git、IDE、单元测试等开发工具。
2. **进阶实战（2-4 个月）**
   - 深入 Spring Boot / Spring MVC 的 IOC、AOP、Starter 机制。
   - 学习关系型数据库设计、索引优化、事务隔离级别与锁模型。
   - 引入缓存、消息队列，理解 CAP、BASE 等分布式理论。
3. **架构提升（长期）**
   - 微服务拆分、API 网关、服务容错、链路追踪与配置中心。
   - 自动化部署、灰度发布、容器化与 K8s 资源管理。
   - 安全体系、性能压测、可观测性建设。

## 学习策略

- **目标驱动**：围绕业务场景设定学习目标，避免只追热点。
- **知识图谱**：把新知识挂到已有知识树上，形成联想记忆。
- **多练多复盘**：每个阶段至少实现一个端到端的小项目，并在项目总结中梳理问题和解决方案。
- **关注源码与原理**：通过阅读 Spring、MyBatis 等框架源码理解设计模式和实现细节。

## 实践建议

- 从单体应用起步，在实际需求中引入缓存、消息队列，再考虑拆分微服务。
- 学会编写自动化测试与脚本，把重复操作工具化。
- 关注线上指标（QPS、P99、错误率、可用性），锻炼互因分析能力。

## 自查清单

- 是否能独立设计数据库表结构并解释冗余与范式权衡？
- 是否理解常见架构（单体、微服务、Serverless）的选择原则？
- 是否掌握日志分级、链路追踪与监控报警的组合搭建？
- 是否针对并发、高可用、数据一致性提出应对策略？
