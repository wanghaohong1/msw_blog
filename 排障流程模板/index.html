              # 排障流程模板

              适用于紧急故障响应场景，确保排查步骤有序可追溯。

              # 1. 告警评估（0-5 分钟）

              • 核实告警信息：触发时间、指标、影响范围。
              • 判断故障等级：是否影响核心链路或大量用户。
              • 建立沟通频道：创建应急群/会议，指定主责人与记录人。

              # 2. 快速止血（5-15 分钟）

              • 评估是否可立即限流、降级或回滚。
              • 收集关键数据：指标面板截图、日志异常数量。
              • 若影响可控，执行止血方案并同步相关方。

              # 3. 深入定位（15-45 分钟）

              • 对照最近变更，优先排查新上线功能或配置。
              • 使用三板斧：指标 → 日志 → 依赖链路。
              • 必要时抓取 dump、线程栈或开启 Debug 日志。

              # 4. 方案评审（45-60 分钟）

              • 汇总定位结果，列出可行方案、影响面与风险。
              • 主责人召集业务、运维确认执行步骤与回退策略。
              • 在操作前再检查一次依赖服务状态。

              # 5. 执行恢复（60+ 分钟）

              • 严格按照操作手册执行，过程实时播报关键节点。
              • 执行后观察指标、日志，确认恢复成功。
              • 确认无反弹后通知解除告警，恢复正常状态。

              # 6. 复盘与沉淀

              • 记录时间线：告警、响应、定位、恢复各时间点。
              • 分析根因：需求缺陷、代码问题、依赖故障、流程问题。
              • 输出复盘报告并跟踪改进项直至关闭。
              • 同步更新知识库、告警策略与自动化脚本。
